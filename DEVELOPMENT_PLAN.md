## 📦 模块 1：天气 Widget

### 目标
在首页添加天气信息卡片，展示当前天气和未来预报。

### 功能需求
1. **实时天气**：温度、天气状况、图标
2. **预报信息**：未来 3-7 天简要预报
3. **地理位置**：城市名称、自动检测（可选）
4. **刷新机制**：30分钟缓存，手动刷新按钮

### 技术方案
**前端**：
- 组件：`WeatherWidget.tsx`
- API 客户端：`web/src/api/weather.ts`
- 数据源：OpenWeatherMap API（免费版）
- 缓存策略：30分钟 staleTime

**后端**（可选，代理方案）：
- Handler：`server/internal/handler/weather.go`
- 作用：隐藏 API Key，避免前端暴露
- 路由：`GET /api/v1/weather?city={city}`

**配置**：
- API Key 存储：环境变量 `OPENWEATHER_API_KEY`
- 默认城市：配置文件 `config/app.yaml`

### 实现步骤
1. 前端创建 WeatherWidget 组件
2. 集成 OpenWeatherMap API
3. 添加到首页，支持拖拽排序
4. （可选）后端代理 API 请求

### 验证点
- [ ] 天气数据正确显示
- [ ] 图标和样式符合设计系统
- [ ] 缓存机制正常工作
- [ ] 手动刷新功能正常

### 预估工时
- 前端：2-3 小时
- 后端代理（可选）：1 小时
- 总计：**3-4 小时**

## 📦 模块 2：番茄钟 Widget

### 目标
实现专注计时器，帮助用户管理时间和提升专注力。

### 功能需求
1. **计时功能**：25分钟专注 + 5分钟休息
2. **状态切换**：开始、暂停、重置
3. **进度显示**：环形进度条 + 剩余时间
4. **通知提醒**：计时结束时浏览器通知
5. **统计记录**：今日完成的番茄数

### 技术方案
**前端**：
- 组件：`PomodoroWidget.tsx`
- 状态管理：React useState + useEffect
- 计时器：`setInterval`（标准库）
- 通知：Notification API（浏览器原生）
- 存储：localStorage（记录今日统计）

**后端**：
- 无需后端（纯前端实现）

**配置**：
- 工作时长：25 分钟（可自定义）
- 休息时长：5 分钟（可自定义）
- 长休息：15 分钟（每 4 个番茄后）

### 实现步骤
1. 创建 PomodoroWidget 组件
2. 实现计时逻辑和状态管理
3. 设计环形进度条 UI
4. 添加浏览器通知功能
5. 实现今日统计（localStorage）
6. 集成到首页，支持拖拽排序

### 验证点
- [ ] 计时精确，无漂移
- [ ] 状态切换流畅（开始/暂停/重置）
- [ ] 进度条动画流畅
- [ ] 通知正常弹出
- [ ] 统计数据准确保存

### 预估工时
- UI 组件：2 小时
- 计时逻辑：1 小时
- 通知和统计：1 小时
- 总计：**4 小时**

## 📦 模块 3：通知中心 Widget

### 目标
聚合所有模块的通知消息，提供统一的信息入口。

### 功能需求
1. **消息聚合**：RSS 新文章、GitHub 事件、服务状态变更
2. **消息列表**：时间倒序，显示最近 20 条
3. **标记已读**：点击消息标记为已读
4. **清空功能**：一键清空所有消息
5. **角标提示**：未读消息数量

### 技术方案
**前端**：
- 组件：`NotificationWidget.tsx`
- 数据结构：`Notification { id, type, title, message, time, read }`
- 存储：localStorage（简单方案）

**后端**（可选，高级方案）：
- 表：`notifications` 表
- Handler：`server/internal/handler/notification.go`
- 路由：`GET /api/v1/notifications`, `PUT /api/v1/notifications/{id}/read`

**消息来源**：
- RSS：新文章抓取成功时生成通知
- GitHub：有新事件时生成通知
- 服务监控：状态变更时生成通知
- 番茄钟：完成时生成通知

### 实现步骤
1. 定义通知数据结构
2. 创建 NotificationWidget 组件
3. 实现消息列表和已读标记
4. 集成到各个模块（RSS、GitHub等）
5. 添加到首页，支持拖拽排序
6. （可选）后端持久化

### 验证点
- [ ] 消息正确聚合
- [ ] 标记已读功能正常
- [ ] 清空功能正常
- [ ] 角标数量准确
- [ ] 与其他模块集成无误

### 预估工时
- 前端组件：2-3 小时
- 集成到现有模块：2 小时
- 后端持久化（可选）：2 小时
- 总计：**4-7 小时**
