# 基金功能增强

## Goal

优化基金功能的用户体验：
1. 首页和详情页展示最后的基金净值获取时间
2. 各统计图在刷新数据后实时更新
3. 增加每日盈亏统计图

## Background

当前基金功能存在以下问题：
- 首页 `FundWidget` 和详情 `FundPage` 均未显示 `last_updated` 时间，用户不知道看到的数据是何时获取的
- "当日盈亏"面板刷新后，图表数据没有同步更新（因为图表依赖的 queryKey 与刷新按钮的 invalidation 不一致）
- 缺少每日盈亏的历史走势图，无法直观看到每天赚了多少/亏了多少

## Requirements

### 1. 显示数据刷新时间

- **首页 FundWidget**：在底部或总览区域显示最近更新时间
  - 例如："数据更新：今天 15:30" 或 "数据更新：2026-07-06"
  - 使用相对时间（今天/昨天/具体日期）
- **详情 FundPage**：在统计卡片区域或当日盈亏面板显示
  - 总盈亏卡片下方显示 "数据更新于 HH:mm"
  - 单只持仓行显示 `last_updated`

### 2. 图表实时刷新

- 点击"刷新当日盈亏"后，确保图表自动更新：
  - `queryClient.invalidateQueries` 需要覆盖所有 `fund` 开头的 queryKey
  - 图表组件使用 useQuery 时确保使用了正确的 queryKey
- 当日盈亏面板刷新后，持仓列表的 `current_nav` 也应更新

### 3. 每日盈亏统计图

- 在统计图表区域新增"每日盈亏"柱状图
- x 轴：日期（近 N 天）
- y 轴：当日盈亏金额（元）
- 红色柱子表示盈利，绿色柱子表示亏损
- 数据来源：利用 `fund_nav_history` 表计算每日市值变化 → 每日收益
- 后端新增 API 或扩展现有 API 返回每日盈亏数据

## Acceptance Criteria

- [ ] FundWidget 显示最后更新时间
- [ ] FundPage 统计卡片显示最后更新时间
- [ ] FundPage 持仓列表显示最后更新时间
- [ ] 点击刷新后图表数据同步更新
- [ ] 新增每日盈亏柱状图
- [ ] 每日盈亏图数据准确（来自净值历史记录）
