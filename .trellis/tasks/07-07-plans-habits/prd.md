# 目标管理中心：待办习惯整合

## 需求概述

将现有的 `/plans`（计划管理）和 `/check-in`（习惯打卡）两个独立页面合并为一个统一的**目标管理中心**，让用户在一个页面内完成"创建目标→查看待办→每日打卡→追踪进度"的完整流程。

## 功能范围

### P0 - 核心改造
1. **页面合并**：将打卡功能嵌入 PlansPage，成为一个 section
2. **进度总览卡片**：环形图/进度条展示整体习惯完成率
3. **日历增强**：支持补打卡、取消打卡
4. **习惯详情面板**：点击习惯查看详细追踪数据

### P1 - 体验增强
5. **TodoWidget 增强**：同时展示待办和今日待打卡习惯
6. **管理后台扩展**：统一管理

### P2 - 未来可扩展
7. **习惯分类**：健康/学习/工作等分组

## 用户体验流程

```
用户进入 /plans
  ├── 看到待办看板（三列优先级）
  ├── 下方看到「今日打卡」区 → 一键打卡
  ├── 下方看到「进度总览」→ 习惯完成率 + 本周/月统计
  ├── 下方看到「打卡日历」→ 可切换月份查看/补打卡
  └── 点击某个习惯 → 弹出详情面板 → 查看趋势/编辑
```

## 约束

- 保留 `/check-in` 路由作为快捷入口，不破坏已有书签/链接
- 后端 API 尽量复用现有，仅按需新增
- 遵循 Ponytail 原则：标准库优先，零不必要依赖
