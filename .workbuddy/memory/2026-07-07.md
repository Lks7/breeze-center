# 2026-07-07

## 修复：目标管理中心 (PlansPage) 添加待办报 500

**根因**：`server/internal/store/schema.go` 的 `migrate()` 把多条 `ALTER TABLE` 放在一次 `db.Exec` 里执行。
`fund_holdings` 的三列（`target_profit_rate`/`stop_loss_rate`/`alert_triggered`）已写进 `CREATE TABLE`，
于是第一条 `ALTER ... ADD COLUMN` 触发 "duplicate column name"，**单次多语句 Exec 遇错即中断**，
导致后面的 `todos` 三列（`is_habit`/`habit_frequency`/`habit_target`）从未被加进真实表。
store 的 `Create`/`List`/`get` 又引用这三列 → SQLite 报错 → 后端返回 500。

**修复**：把 `migrate()` 改为逐条执行 `ALTER`，并对每条单独忽略 "duplicate column name" 错误（幂等）。
重新 `go build -o bin/server.exe ./cmd/server`，杀掉旧进程 (PID 23044) 并重启后端。

**验证**：`POST /api/v1/admin/todos` 普通待办与习惯待办均返回 201；`todos` 表现在含 `is_habit/habit_frequency/habit_target`；`GET` 列表正常。已清理测试数据（count 回到 4）。

**教训（可复用）**：modernc.org/sqlite 的 `db.Exec` 多语句遇首个失败即中止；增量迁移必须逐条执行并各自容忍 "duplicate column name"。
